## Краткое описание
Этот репозиторий собирает наглядные и читаемые реализации алгоритмов сжатия и кодирования, которые удобно изучать, тестировать и расширять. Код ориентирован на простоту и понятность, а не на максимальную производительность.

## Реализованные алгоритмы
(Если какой-то файл/модуль отсутствует в репозитории, этот список можно отредактировать.)
- RLE (Run-Length Encoding)
- Хаффман (Huffman coding)
- Арифметическое кодирование (Arithmetic coding)
- LZW (Lempel–Ziv–Welch)
- Шеннон–Фано (Shannon–Fano)
- Эллиасовы коды (Elias gamma/delta)
- Вариантный байтовый код (Variable-byte)
- Утилиты для оценки: коэффициент сжатия, замер времени

## Установка
1. Клонируйте репозиторий:

   git clone https://github.com/Luk4red/Algorithms-for-encoding-and-compressing-information.git
   cd Algorithms-for-encoding-and-compressing-information

2. (Опционально) Создайте и активируйте виртуальное окружение:

   python -m venv .venv
   source .venv/bin/activate   # macOS/Linux
   .\.venv\Scripts\activate    # Windows

3. Установите зависимости (если есть):

   pip install -r requirements.txt

Если dependencies отсутствуют — этот шаг можно пропустить.

## Примеры использования
Ниже — шаблон использования. Замените имена модулей и классов на реальные из репозитория.

Пример: кодирование и декодирование с использованием гипотетического класса HuffmanCodec

```python
from algorithms.huffman import HuffmanCodec

codec = HuffmanCodec()
raw = b"пример текста для сжатия"
encoded = codec.encode(raw)
decoded = codec.decode(encoded)
assert decoded == raw
print(f"Исходный размер: {len(raw)} bytes")
print(f"Размер после сжатия: {len(encoded)} bytes")
```

Пример использования CLI (если в репозитории есть скрипты examples/compress.py и examples/decompress.py):

```bash
# Сжать файл
python -m examples.compress --algo huffman input.txt output.huf

# Разжать файл
python -m examples.decompress --algo huffman output.huf restored.txt
```

## Тесты
Положите тесты в папку tests/ и запускайте через pytest:

```bash
pip install -r requirements.txt   # если pytest добавлен в зависимости
pytest tests/
```

Рекомендуется покрыть тестами корректность циклов encode->decode и граничные случаи (пустые входы, все одинаковые байты, уникальные символы и т.д.).

## Сравнение и производительность
Рекомендуется добавить скрипты для автоматического сравнения алгоритмов по:
- Коэффициенту сжатия (compressed_size / original_size)
- Времени кодирования и декодирования (time.perf_counter / timeit)
- Потреблению памяти (tracemalloc или psutil)

Примерный формат отчёта: таблица алгоритмов × наборов данных с колонками: сжатие, скорость кодирования, скорость декодирования.
